Bug Reports & Contributing

Block-Suite is actively developed. If you find a bug, experience unexpected behavior, or have a feature request, please report it on GitHub so it can be addressed promptly.

How to Report a Bug

1
Go to GitHub Issues
Open github.com/itsreyi/BlockSuite/issues and click New Issue.
2
Use a clear title
Summarize the issue in a few words. Good: "Velocity fails to start when target port is above 60000". Bad: "doesn't work".
3
Fill in the details
Use the template below. The more info you provide, the faster the bug can be reproduced and fixed.
4
Submit & follow up
After creating the issue, watch for comments asking for clarifications.

Common Issues & Solutions

Velocity won't start

  • Ensure Java 17+ is installed and in your system PATH
  • Check if another process is using port 25577 (default Velocity bind port)
  • Look for errors in runtime/velocity/logs/latest.log
  • Click Setup Environment again to regenerate configs

Players can't connect / get disconnected

  • Verify the target server is online using the Status button
  • Check the forwarding mode — try none first, then legacy if needed
  • If using playit.gg, ensure the tunnel is active and the claim URL was completed
  • Try disabling HAProxy Protocol if players drop immediately on connect
  • Enable ViaVersion if the client version doesn't match the target

Server status shows "Offline" but the server is running

  • Double-check the IP address and port in Server Cloning tab
  • The target may have a firewall blocking status pings — try with a known public server first
  • Some servers use SRV records; try the numeric IP directly

Playit.gg tunnel not working

  • The first time you start a tunnel, a claim URL appears in the Activity Log — you must open it in a browser and claim the tunnel
  • Check if runtime/playit/playit.exe was downloaded successfully
  • Ensure no firewall is blocking outgoing connections to playit.gg

Update check fails / "Copy Error" appears

  • Block-Suite checks https://api.github.com for new releases
  • Ensure you have internet connectivity
  • If the error persists, click Copy Error to copy the technical details and include them in your bug report

"Port already in use" error

  • A previous Velocity instance may still be running. Open Task Manager and end any java.exe processes.
  • Another application may be using port 25577. Change the port in Velocity's config or stop the other app.

Security & Responsible Use

🔒 Authorized Testing Only

Block-Suite is designed for authorized security assessments of Minecraft servers. Only use it on servers you own or have explicit written permission to test. Unauthorized use may violate laws in your jurisdiction.

If you discover a security vulnerability in Block-Suite itself, please report it privately via GitHub's Security Advisory feature rather than opening a public issue.
